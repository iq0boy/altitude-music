## Commands

```bash
npm run dev       # Astro dev server at localhost:4321
npm run build     # Static build to ./dist/ (also generates sitemap-index.xml)
```

Node >= 22.12.0 is required.

## Architecture

Static-rendered (SSG) Astro site for the Altitude Music studio in Louvain-la-Neuve. Three locales (`fr`, `en`, `nl`) routed via Astro's built-in i18n (`prefixDefaultLocale: true` ŌĆö every page lives under `/fr/`, `/en/`, or `/nl/`). The root `/` redirects to `/fr/`.

### Rendering strategy: islands

Static, zero-JS sections are `.astro` components: `Nav`, `Hero`, `Marquee`, `Services`, `About`, `Blog`, `Footer`. They render at build time and ship no JavaScript.

Interactive sections are React islands (`.tsx`) hydrated only when needed:

| Component | Hydration | Why |
|---|---|---|
| `Portfolio.tsx` | `client:visible` | Genre filter, fake play simulation, sticky now-playing bar |
| `Videos.tsx` | `client:visible` | Active video swap |
| `Testimonials.tsx` | `client:visible` | Filter + submit form |
| `Booking.tsx` | `client:visible` | Calendar, slot selection |
| `Contact.tsx` | `client:visible` | Form + mailto |
| `ChatWidget.tsx` | `client:idle` | Always-floating WhatsApp widget |
| `ServiceDetail.tsx` | `client:load` | Listens to `hashchange` for `#service/<key>` modal routing ŌĆö must be live before user interaction |

Add new interactive sections as `.tsx` islands. Add new static sections as `.astro` components. Keep `client:load` only for things that respond to URL state on first paint.

### Content collections

Defined in `src/content.config.ts` (Astro v6 location ŌĆö **not** the legacy `src/content/config.ts`). Two collections, both using the `glob` loader:

- **`blog`** ŌĆö one markdown file per language: `<slug>.<lang>.md`. Frontmatter must include `lang` and `postSlug` (we use `postSlug` instead of `slug` because the glob loader uses the `slug` field internally as the entry id, which would collide across languages). Pages are generated by `[lang]/blog/[slug].astro` from `getCollection('blog')`.
- **`portfolio`** ŌĆö one markdown per track. The `TRACKS` array lives in markdown frontmatter, not a TS file. Sorted by the `sortOrder` frontmatter field.

To render markdown body, import `render` from `astro:content` and call `render(entry)` (the v5 `entry.render()` does not exist on glob-loaded entries).

### Data shape

- `src/i18n/{fr,en,nl}.json` ŌĆö UI strings. Loaded statically via `src/i18n/utils.ts` (`getTranslations(lang)`). The shape is mirrored across all three files ŌĆö when you add a new key to one, add it to all three.
- `src/data/services.ts` ŌĆö `SERVICE_KEYS`, `DEFAULT_PRICES`, `SERVICE_GALLERY`, `SERVICE_VIDEOS`, `SERVICE_LONG`. Service descriptions in three languages live here (not in i18n JSON) because they're long-form prose tied to service identity.
- `src/data/team.ts`, `videos.ts`, `testimonials.ts` ŌĆö multilingual content with shape `{ fr, en, nl }` per field.

### Styling

`src/styles/global.css` is the single source of truth, imported once by `BaseLayout.astro`. CSS custom properties drive theming via HTML attributes set on `<html>` in the layout: `data-theme` (`dark`/`light`), `data-density` (`compact`/`regular`/`comfy`), `data-anim` (`on`/`off`). Custom fonts (Satoshi, Clash Display) live in `public/fonts/`; Google Fonts are pulled via `<link>` in the layout.

### SEO

`BaseLayout.astro` emits canonical URL, OG tags, hreflang links for all three locales (+ `x-default`), and a `MusicRecordingStudio` JSON-LD block. The site URL is hard-coded as `https://altitudemusic.be` in `astro.config.mjs` and the layout. The sitemap integration auto-generates per-locale sitemaps.

### Scroll reveal

Elements with `class="reveal"` get a `.in` class added when they enter the viewport, via a small `IntersectionObserver` script at the bottom of `BaseLayout.astro`. No React needed for this.

### Legacy code

`website/` contains the original single-file React-via-CDN prototype. It is **not** part of the build. Keep it for reference only ŌĆö make all changes under `src/`.
